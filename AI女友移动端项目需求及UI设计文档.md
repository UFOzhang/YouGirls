|项目名称|AI女友-移动端情感陪伴交互应用|开发框架|Vue3 + Vite + Vant 4|
|---|---|---|---|

# 二、项目需求分析

## 2.1 核心需求

1. **拟人化聊天交互**：支持文字、语音两种交互方式，AI回复需贴合所选人设风格，具备上下文理解能力；聊天记录支持本地存储、删除、清空。

2. **多风格人设切换**：提供温柔系、可爱系、御姐系三类预设人设，支持定制系人设（自定义昵称、头像、性格标签），切换后实时同步至聊天场景。

3. **语音交互功能**：实现语音输入（语音转文字）、语音输出（文字转语音），支持语速、音量调节，可开启/关闭语音播报。

4. **移动端适配体验**：适配主流iOS/Android手机屏幕，界面响应式布局，操作流程简洁，支持触摸反馈、手势操作（如返回、下拉刷新）。

## 2.2 次要需求

1. **个人中心管理**：展示聊天统计（累计聊天次数、时长）、清除缓存、查看版本信息等功能。

2. **首次启动引导**：引导用户选择初始人设、开启语音权限、简要介绍核心功能。

3. **消息状态反馈**：显示消息发送中、发送成功、AI思考中三种状态，提升交互清晰度。

4. **异常处理机制**：网络异常、API调用失败时显示友好提示，支持重试操作。

## 2.3 非功能性需求

- **性能要求**：首屏加载时间≤3秒，聊天消息发送/接收响应时间≤1秒，语音识别/合成延迟≤2秒。

- **兼容性要求**：支持iOS 12及以上、Android 8.0及以上系统，适配屏幕尺寸320px-428px。

- **安全要求**：本地存储数据加密处理，语音数据传输采用HTTPS协议，不收集敏感用户信息。

- **易用性要求**：操作路径简洁，核心功能（聊天、切换人设）点击次数≤2次，界面风格统一易识别。

# 三、UI设计规范

## 3.1 设计风格定位

整体采用**清新柔和风**，贴合情感陪伴场景，以粉色系为主色调（传递温馨感），搭配白色、浅灰色作为中性色，各功能模块通过颜色区分，视觉层次清晰。界面元素采用圆角设计，搭配轻微阴影，增强立体感和触摸友好性。

## 3.2 色彩规范

|颜色类型|色值|应用场景|

## 3.4 组件规范

- **按钮**：高度统一为56px（小屏适配52px），圆角28px，带4px阴影；默认状态有渐变背景，点击时缩放至0.98倍，阴影变浅。

- **卡片**：圆角12px，内边距16px，阴影0 4px 12px rgba(0,0,0,0.08)；选中时上浮4px，阴影增强至0 8px 16px rgba(255,107,139,0.15)。

- **输入框**：高度56px，圆角28px，内边距0 24px，边框1px solid #e8f4f8，背景色#ffffff。

- **头像**：预设人设头像尺寸80px×80px（小屏70px×70px），圆角50%，边框3px solid #ffffff，带轻微阴影。

- **标签**：小尺寸，圆角12px，内边距2px 6px，字体12px；主标签用主色调，辅助标签用中性色。
